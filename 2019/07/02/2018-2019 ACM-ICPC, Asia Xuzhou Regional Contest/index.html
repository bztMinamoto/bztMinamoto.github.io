C Rikka with Consistency

题面：$n$个点组成一段折线，折现两端(均为折线最低点)有两个人，要求两人每时每刻都必须处于同一高度，求两人走到对方初始位置的最短距离和。（保证有解）

做法：对于每一个点在其他线段上的$y$轴交点算出来，然后两边开始搜索就可以了，每次要么往回走，要么往前走，要么不走。

D Rikka with Subsequences

题面：给你一个长度为$n$的序列$1<=a_i<=n$,以及一个$n\ast n$的01矩阵$M_{ij}$。一个子序列${a_{p_i}}$被称为Yuta序列当且仅当所有的$M_{a_{p_{i-1}}a_{p_{i}}}=1$,求所有不同的Yuta序列的出现次数的立方和。$n<=300,T<=20$

做法：类似于NOI的管道取珠的做法，把立方和理解为三个操作序列，然后设计个普通的$dp$，$dp[i][j][k]$表示第一个到了$i$，第二个到了$j$，第三个到了$k$。因为三个是同时进行的，所以$k$是可以省掉的。发现这个$dp$可以前缀和优化，所以就做完了。

E Rikka with Data Structures

题面：给出一个序列，有三个操作。第一种是区间加法，第二种是区间覆盖，第三种是给出$x$和$[l,r]$，求出$y$在$[l,r]$的个数满足$max(a[min(x,y)],a[min(x,y)+1],…,a[max(x,y)])=max(a[x],a[y])$。

做法：理性感觉一下，这好像就是线段树，主要是第三个操作有点奇怪。考虑把每个位置看成一个二维平面上的点，横坐标为这个位置的下标，纵坐标为这个位置的值。那么第三个操作我们就可以把$x$和$[l,r]$区间内的所有点相连，这是一张折线图。

想一想哪些点是合法的。如果一个点和$x$连接的折线中，有一个点比$x$和这个点都要高，那么这个点就是不合法的。也就是说，一个点合法的话要么在这段折线中它是最高的，要么$x$是最高的。这个最高就是最大值，可以用线段树维护。

那么点的个数呢？这其实是很好想的。在一个折线段中，如果一直是单调递增或单调递减而且最小值比前面折线的任何点都要高，那么这一段都是合法的。也就是说，我们可以维护这样一个折线段的长度。

我们用$ln$和$rn$分别表示这个点左边和右边单调折线段的长度。每次统计答案的时候，找到最低的且比前面那个最高点还要高的点的下标，这样折线段的长度就维护出来了，于是用线段树维护一下就做完了。

G Rikka with Intersections of Paths

题面：给出一棵$n$个点的树，再给出$m$条树上的链。现在需要选出$k$条链，那么就有$C(m,k)$种选择链的方式。若一种选择方式中$k$条链有一个公共交点，则贡献加一。问最终贡献。

做法：其实我们能发现，一条边若被覆盖了$s$次，则这条边连接的两个端点也都被覆盖了至少$s$次。那么也就是说，点的贡献会比边的贡献多一样。所以我们只用把点的贡献减去边的贡献就是最终答案了。

I Rikka with Sorting Networks

题面：一个排序网络被描述如何：对于一个长度为$n(n\leq 50)$的数列，尝试最多$10$次指定的比较（比较某$2$个指定的位置），如果比较的结果是$>$，就交换着两个位置的权值。问有多少$1$到$n$的全排列，满足经过这个排序网络后，会形成一个最长上升子序列长度是$n-1$的序列。

做法：最终结果的序列最多$n^2$个，于是枚举最终的状态，倒着推回去。

J Rikka with An Unnamed Temple

题面：给出一张$n$个点$m$条边的$DAG$，每一个点有一个重量$w$和价值$c$。给出一个$s$和$t$，现要求对于每一个点$i$，找出一条不包括$i$的$1$到$n$的简单路径，使其路径上的点的重量和模$s$==$t$，并且价值最大。输出这个价值和方案数，找不到输出$-1$。

做法：随便搞出一个拓扑序，然后按这个拓扑序分别前缀后缀地做个背包。考虑对于一个点$i$不经过的情况，则枚举一个点然后跨过这个点$i$做一个背包合并。由于每一个点$i$都做一遍会超时，那么就对于一次跨越的所有点都同时做个背包合并，这用线段树维护一下就可以了。时间复杂度$O(mlogn+ns+ms)$。

L Rikka with Grid Graphs

题面：给出一张$n*m$的格点无向图，要求对每一条边定向，求最终为$DAG$的方案数。

做法：说到无环定向数，第一反应就是$Tutte$多项式。不过好像这题没用到那个东西。做法是暴力状压每一行的定向状态，然后用$map$存取整个状态拿来$DP$即可。复杂度大概是$O(2^{2n}\ast n^3)$（假设$(n,m)$同阶）。
